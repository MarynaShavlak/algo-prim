**Responsibilities.**

- **`core.py`** does not depend on any plotting libraries — it is the pure algorithm on top of `networkx`, importable anywhere. It contains `prim_mst` (the basic lecture implementation — the one the README walks through line by line), `prim_mst_steps` (the same logic + an **event journal** for the visualizations), `prim_mst_eager` (the bonus duplicate-free-queue version), `prim_msf` (a minimum spanning forest for disconnected graphs), `build_graph` and `mst_weight`.
- **`style.py`** is the single source of the visual theme: the colour palette as named constants, `FIGURE_DPI`, fonts, and `configure_style()`. The look of every figure can be changed from one place.
- **`i18n.py`** — bilingual labels: `t()` returns a string in the current language (`uk` by default / `en`), `set_lang()` switches it. The dictionary key is the Ukrainian string itself, so the Ukrainian figures stay byte-for-byte unchanged. Every visible label in `visualization.py` and the examples is wrapped in `t()`.
- **`visualization.py`** (requires `matplotlib` and `networkx`) takes the palette from `style.py` and draws everything stateful: the graph at any moment of the algorithm (`draw_graph_state`), the priority-queue panel (`draw_queue_panel`), the main step frame `[graph | queue]` (`draw_prim_step`), the evolution grid, the result with the rejected edges, the cut property (`draw_cut`) and the cable analogy. Textual step summaries — `step_summary`/`show_step`.
- **`animation.py`** stitches frames into a looped **GIF** (via Pillow, always) and additionally encodes the same frame sequence into an **MP4** (via `ffmpeg`, when available). GIF is the guaranteed inline-viewable format; MP4 is the lighter file with playback controls.
- **`walkthrough.py`** builds the step-by-step code ↔ graph ↔ queue panels: the snapshot journal (`build_steps`) with a code-line highlight map, the code panel (`draw_code_panel`) and the compositors (`render_code_step`, `draw_code_walkthrough_grid`); the state panels reuse `draw_graph_state` and `draw_queue_panel`. The colour of a code line encodes the branch that fired (active / accepted / skipped), and every popped edge yields two frames: "heappop — checking" and "the branch fired".
- **`examples/_common.py`** removes duplication across the examples: the `Agg`/path setup, `save_figure`, `save_anim`, the MST console summary (`report_mst`) and the `GraphExample` dataclass holding an edge list together with vertex coordinates.
- **`examples/_graphs.py`** — the single source of truth for the three teaching graphs (`ABCDEF`, `ABCDEFG`, `ISLANDS`) and the cable-analogy plans; every example imports them, so the edges and coordinates are defined exactly once.
- The **`examples/`** scripts merely take the graph they need from `_graphs.py`, call the algorithm and the drawing functions, and save the figures.
